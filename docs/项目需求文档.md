# 项目需求文档：Visual Superpowers (可视化智能体工作台)

## 1. 项目愿景 (Vision)

将原本基于终端和 Markdown 的 `superpowers` 工作流升级为“数据驱动 + 视觉交互”的平台。

* **输入：** 用户的自然语言指令。
* **逻辑：** 遵循 `superpowers` 的设计模式（Socratic questioning, TDD, Systematic planning）。
* **输出：** 弃用纯 Markdown，改为产出 **结构化 JSON**。
* **交互：** 借鉴 `open-slide` 的架构，通过 **React 实时渲染 JSON** 为可视化界面（思维导图、看板、甘特图），并支持从 UI 反馈数据给 Agent。

## 2. 核心架构设计 (Architecture)

Agent 需要理解并实现以下三个层次：

### A. 逻辑层 (The Brain - 借鉴 superpowers)

* **Skills 系统：** 保持 `superpowers` 的原子化技能划分。
* **状态管理：** Agent 的工作成果必须持久化为 `state.json` 而不是 `TODO.md`。

### B. 数据协议层 (JSON Bus)

* **Strict Schema：** 所有技能输出必须符合特定的 JSON Schema（使用 TypeScript 定义）。
* **双向同步：** * `Agent -> JSON`：Agent 修改本地文件。
* `UI -> JSON`：前端通过文件监听或 API 修改 JSON。
* `JSON -> Agent`：Agent 读取 JSON 状态作为下一轮对话的 Context。



### C. 表现层 (The Canvas - 借鉴 open-slide)

* **固定比例渲染：** 借鉴 `open-slide` 的 1920x1080 固定画布，确保排版一致性。
* **动态组件：** 根据 JSON 中的 `skill_type` 自动加载对应的可视化组件。

---

## 3. 分阶段执行计划 (Implementation Roadmap)

### 第一阶段：基础框架搭建 (Foundation)

1. **工程初始化：** 参考 `open-slide` 的 monorepo 结构，建立 `apps/web` (React) 和 `packages/skills` (Agent logic)。
2. **CLI 桥接：** 实现一个本地监听器，当 `state.json` 变化时，前端页面自动热更新（Hot Reload）。

### 第二阶段：可视化头脑风暴 (Visual Brainstorming)

1. **移植逻辑：** 将 `superpowers/brainstorm` 的提问逻辑改写，使其产出包含节点关系、状态（探讨中/已采纳/已否决）的树状 JSON。
2. **实现 UI：** 开发一个基于 React 的思维导图组件。
3. **闭环功能：** 用户点击 UI 上的节点，自动生成一段“用户反馈信息”写入临时文件，Agent 读取后针对该节点继续追问。

### 第三阶段：执行计划可视化 (Visual Planner)

1. **数据转换：** 将 `superpowers` 产出的执行步骤转化为包含 `id`, `dependency`, `status`, `progress` 的 JSON 数组。
2. **实现看板/进度条：** 在前端实时显示当前 Agent 正在执行哪一步（Doing），哪些已完成（Done）。

---

## 4. 给 Agent 的技术指令 (Technical Instructions)

> **请 Agent 遵循以下开发原则：**

1. **Context 优先：** 你必须深入阅读本地 `open-slide` 仓库中的渲染引擎逻辑，以及 `superpowers` 仓库中的 prompts 设计。
2. **Schema 驱动：** 在开始编写任何代码前，先为该技能定义 TypeScript 接口。
3. **模块化：** 新增技能应作为独立的插件放入 `skills/` 目录，前端组件应具有高度复用性。
4. **UI 交互：** 实现类似 `open-slide` 的“注释反馈机制”，让用户在网页上的点击能精准映射到源码或 JSON 数据块上。
